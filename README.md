# DevSpace

A parallel development project manager for macOS. Run multiple projects simultaneously with unique hostnames (`myapp.test`, `api.test`) — each gets its own browser origin, cookies, and OAuth redirect URIs. No more remembering port numbers.

## How It Works

1. **Register a project** — `devspace init` in any project directory
2. **Start your dev server** — however you normally do (`npm run dev`, `cargo run`, etc.)
3. **Auto-detected** — DevSpace watches for new listening ports and maps them to `yourproject.test`
4. **Access via hostname** — visit `https://yourproject.test` in your browser

Under the hood:
- A **Rust daemon** watches for listening TCP ports and matches them to registered projects via process working directory
- **Caddy** reverse-proxies traffic with automatic HTTPS (internal CA)
- A built-in **DNS responder** resolves `*.test` to `127.0.0.1`
- A native **macOS menu bar app** manages windows, hotkeys, colored borders, and layout tiling across projects

## Install

### Build from Source

```bash
git clone https://github.com/yourusername/DevSpace.git
cd DevSpace
bash scripts/build.sh
```

This produces `build/DevSpace.app`. Drag it to `/Applications`.

Optionally install the CLI to your PATH:

```bash
ln -sf /Applications/DevSpace.app/Contents/Helpers/devspace /usr/local/bin/devspace
```

### First-Time Setup

```bash
devspace setup
```

This runs one-time system configuration (requires sudo):
- Creates `/etc/resolver/test` for DNS resolution
- Installs Caddy's root CA for trusted HTTPS
- Sets up port forwarding (80 → 8080, 443 → 8443)

Caddy is auto-downloaded on first run if not already installed.

## Usage

The menu bar app launches the daemon automatically. For CLI usage:

```bash
# Start the daemon manually (if not using the app)
devspace daemon start

# Initialize a project
cd ~/projects/my-app
devspace init

# Start your dev server as usual
npm run dev  # or cargo run, python manage.py runserver, etc.

# DevSpace auto-detects the server and creates the route
# Visit https://my-app.test in your browser

# Check status
devspace status
```

## Configuration

Global config at `~/.config/devspace/config.toml`:

```toml
# TLD for project hostnames (default: "test")
# Set to "localhost" to skip DNS setup (access via myapp.localhost:8080)
tld = "test"

[caddy]
http_port = 8080    # Caddy HTTP port
https_port = 8443   # Caddy HTTPS port

[daemon]
log_level = "info"
dns_port = 5553     # DNS responder port
```

Per-project config at `.devspace.toml`:

```toml
[project]
name = "my-app"
hostname = "my-app"
```

## Architecture

```
Browser → https://myapp.test
         ↓ DNS: /etc/resolver/test → daemon DNS responder (:5553)
         ↓ resolves to 127.0.0.1
         ↓ pfctl redirects :443 → :8443
    Caddy (:8443) → reverse_proxy 127.0.0.1:3000
         ↑ Caddyfile auto-generated by daemon
         ↑ Port watcher discovers :3000 belongs to "myapp"
```

### Crates

| Crate | Purpose |
|-------|---------|
| `devspace-core` | Shared types, config, validation |
| `devspace-proto` | JSON-RPC 2.0 message types |
| `devspace-daemon` | Daemon binary — Caddy management, DNS, port watcher, IPC |
| `devspace-cli` | CLI binary — init, status, setup, daemon control |

### macOS App

Native AppKit menu bar app with:
- Window tracking and project association
- Global hotkeys (Ctrl+1-9) for project switching
- Colored border overlays per project
- Layout tiling (code focus, equal split, etc.)
- Mission Control / Spaces integration

## Requirements

- macOS 13+
- Rust toolchain (for building)
- Swift 5.9+ (for building)
